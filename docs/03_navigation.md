## 1. 知识点：机器人导航体系 (Robot Navigation System)

### 1.1 两种地图类型 (Two Map Types)

**度量地图（Metric Map）**：记录每个点的精确坐标，例如激光雷达扫描生成的 2D 占用栅格地图。

> ***Metric Map**: Records precise coordinates for every point, e.g. a 2D occupancy grid generated by lidar scanning.*

```
┌─────────────────┐
│ . . . # # # . . │   # = 障碍物 / obstacle
│ . . . # . . . . │   . = 空闲 / free
│ . . . # . . . . │
│ . . . . . . . . │
└─────────────────┘
```

**拓扑地图（Topological Map）**：记录"地点"（节点）和"路径"（边），不关心精确坐标，只关心哪些地点之间可以走通。

> ***Topological Map**: Records "places" (nodes) and "paths" (edges). Doesn't care about exact coordinates — only which places are reachable from which.*

```
[大厅 / Lobby] ──────── [前台 / Reception]
   │                        │
[电梯厅 / Elevator]      [会议室A / Meeting Room A]
   │
[茶水间 / Pantry]
```

### 1.2 A2 的导航方案 (A2's Navigation Approach)

A2 使用两者的结合：

> *A2 combines both approaches:*

- 建图时用 **SLAM**（同步定位与地图构建）生成度量地图 / **SLAM** (Simultaneous Localization And Mapping) generates a metric map during mapping
- 在度量地图上，人工标记**拓扑导航点**（waypoints）/ **Topological waypoints** are manually labeled on the metric map
- 导航时，用度量地图做障碍物避让，用拓扑图规划路径 / During navigation, the metric map handles obstacle avoidance while the topological graph plans the route

你在代码里操作的就是这些**拓扑导航点**：

> *What you operate in code are these **topological waypoints**:*

```python
points = [
    {"point_id": 101, "name": "前台 / Reception", "pose": {...}},
    {"point_id": 202, "name": "会议室A / Meeting Room A", "pose": {...}},
    {"point_id": 303, "name": "茶水间 / Pantry", "pose": {...}},
]
```

### 1.3 PNC 是什么？ (What is PNC?)

**PNC = Planning & Navigation Control**，负责：

> *PNC = Planning & Navigation Control, responsible for:*

1. 接收目标导航点 / Receiving the target waypoint
2. 规划从当前位置到目标点的路径 / Planning a path from the current location to the target
3. 执行导航（同时调用运动控制走路）/ Executing navigation (calling motion control to walk)
4. 汇报任务状态 / Reporting task status

你的代码只需要和 PNC 服务对话，不需要关心底层的路径规划算法。

> *Your code only needs to communicate with the PNC service; you don't need to worry about the underlying path-planning algorithm.*

## 2. 知识点：任务状态机 (Task State Machine)

PNC 导航任务有明确的状态机：

> *PNC navigation tasks have a well-defined state machine:*

```
下发任务 (Submit task)
    │
    ▼
  IDLE  ──> RUNNING ──────────────> SUCCESS
              │ │
              │ └─> PAUSED ──> RUNNING
              │
              └─> FAILED
```

你的轮询逻辑要处理这 5 种状态：

> *Your polling logic must handle these 5 states:*

```python
while True:
    state = get_nav_state(task_id)

    if state == "PncServiceState_SUCCESS":
        print("到达目标 / Destination reached")
        break
    elif state == "PncServiceState_FAILED":
        print("导航失败 / Navigation failed")
        break
    elif state == "PncServiceState_RUNNING":
        pass   # 继续等待 / Keep waiting
    elif state == "PncServiceState_PAUSED":
        pass   # 被暂停（可能是遇到障碍），等待恢复 / Paused (possibly obstacle), waiting to resume
    elif state in ("PncServiceState_IDLE", ""):
        print("任务异常终止 / Task terminated abnormally")
        break

    time.sleep(1.0)
```

## 3. 知识点：`task_id` 的作用 (The Role of `task_id`)

每次下发导航任务时，你传入一个 `task_id`，服务用它来：

> *Each time you submit a navigation task, you provide a `task_id`. The service uses it to:*

1. 区分不同的导航任务 / Distinguish between different navigation tasks
2. 查询特定任务的状态 / Query the status of a specific task
3. 取消特定任务 / Cancel a specific task

```python
# 下发任务，服务返回确认的 task_id / Submit task; service returns confirmed task_id
task_id = navigate_to(map_id=1, point_id=101, task_id=42)

# 用同一个 task_id 查询状态 / Query status with the same task_id
state = get_nav_state(task_id=42)

# 用同一个 task_id 取消 / Cancel with the same task_id
cancel_nav(task_id=42)
```

**注意 (Note)**：`task_id` 自己管理，通常用递增整数。如果你用了重复的 `task_id`，可能会查到旧任务的状态。

> *You manage `task_id` yourself; typically use an incrementing integer. Using a duplicate `task_id` may return the status of an old task.*

## 4. 涉及的三个服务 (Three Services Involved)

### 4.1 `MappingService.GetCurrentWorkingMap`

```
IP: NAV_IP  Port: 50807
```

获取当前激活的地图 ID / Get the currently active map ID:

```json
{
    "header": {},
    "command": "MappingCommand_GET_CURRENT_WORKING_MAP"
}
```

**请求字段 (Request fields)**：

| 字段 / Field | 含义 / Meaning |
|------|------|
| `header` | 空对象即可，该接口不校验时间戳 / Empty object is fine; this interface doesn't validate timestamps |
| `command` | 固定值，指定这次 Mapping 操作的类型 / Fixed value specifying the type of Mapping operation |

**响应（`GetCurrentWorkingMapRsp`）/ Response**：
```json
{
    "data": {
        "map_id": 3
    }
}
```

| 字段 / Field | 含义 / Meaning |
|------|------|
| `data.map_id` | 当前激活地图的 ID，后续 `GetTopoMsgs` 和 `PlanningNaviToGoal` 都要用这个值 / ID of the active map; needed for `GetTopoMsgs` and `PlanningNaviToGoal` |

### 4.2 `LocalizationService.GetTopoMsgs`

```
IP: NAV_IP  Port: 50807
```

获取指定地图的拓扑导航点列表 / Get the list of topological waypoints for a map:

```json
{
    "header": {},
    "command": "TopoCommand_GET_TOPO_MSG",
    "map_id": 3
}
```

**请求字段 (Request fields)**：

| 字段 / Field | 含义 / Meaning |
|------|------|
| `command` | 固定值，指定这次 Localization 操作类型 / Fixed value specifying the Localization operation type |
| `map_id` | 要查询哪张地图的导航点，从 `GetCurrentWorkingMap` 获取 / Which map to query, obtained from `GetCurrentWorkingMap` |

**响应（`GetTopoMsgsRsp`）/ Response**：
```json
{
    "data": {
        "points": [
            {
                "point_id": 101,
                "name": "前台 / Reception",
                "pixel_pose": {"x": 320, "y": 240}
            }
        ]
    }
}
```

**每个导航点的字段（`PixelNaviPointDataType`）/ Fields of each waypoint**：

| 字段 / Field | 含义 / Meaning |
|------|------|
| `point_id` | 导航点的唯一 ID，`PlanningNaviToGoal` 用这个值指定目标 / Unique waypoint ID; used by `PlanningNaviToGoal` to specify the target |
| `name` | 在建图工具里标注的可读名称 / Human-readable name labeled in the mapping tool |
| `pixel_pose` | 导航点在地图图像上的像素坐标（可视化用，导航时不需要）/ Pixel coordinates on the map image (for visualization; not needed for navigation) |

### 4.3 `PncService`

```
IP: NAV_IP  Port: 53176
```

三个方法 (Three methods):

| 方法 / Method | 作用 / Purpose |
|------|------|
| `PlanningNaviToGoal` | 下发导航任务 / Submit a navigation task |
| `ActionGetState` | 查询任务状态 / Query task status |
| `ActionCancel` | 取消任务 / Cancel task |

### 5.2 下发导航任务 (Submit a Navigation Task)

```python
def navigate_to(map_id: int, target_point_id: int, task_id: int = 1) -> int:
    payload = {
        "header": {
            "timestamp": {"seconds": 0, "nanos": 0, "ms_since_epoch": 0},
            "control_source": 0,   # 0 = 外部控制 / 0 = external control
        },
        "task_id": task_id,
        "map_id": map_id,
        "target_id": target_point_id,
        "guide_line_id": 0,        # 0 = 自动规划路径 / 0 = auto-plan path
        "ackerman_mode": False,    # False = 全向移动，True = 仿车辆转弯 / False = omnidirectional, True = car-like steering
    }
    result = rpc(NAV_IP, 53176, "aimdk.protocol.PncService", "PlanningNaviToGoal", payload)

    if result.get("state") != "CommonState_SUCCESS":
        raise RuntimeError(f"导航下发失败 / Navigation submission failed: {result.get('info', result.get('state'))}")

    return result.get("task_id", task_id)
```

**`PlanningNaviToGoalRequest` 的字段 (Fields)**：

| 字段 / Field | 类型 / Type | 含义 / Meaning |
|------|------|------|
| `header` | object | 请求头，此处时间戳设 0 也可接受 / Request header; timestamp 0 is acceptable here |
| `task_id` | int | 你自定义的任务 ID / Your custom task ID |
| `map_id` | int | 目标点所在地图的 ID / ID of the map containing the target |
| `target_id` | int | 目标导航点的 `point_id` / Target waypoint's `point_id` |
| `guide_line_id` | int | 引导线 ID，`0` 表示不指定（PNC 自动规划）/ Guide line ID; `0` = unspecified (PNC auto-plans) |
| `ackerman_mode` | bool | `false`=全向移动，`true`=仿汽车只能前后+转向 / `false` = omnidirectional, `true` = car-like (forward/backward + steering only) |

**响应（`CommonTaskResponse`）的字段 (Response fields)**：

| 字段 / Field | 含义 / Meaning |
|------|------|
| `state` | `"CommonState_SUCCESS"` 表示任务下发成功（不是到达！）/ Indicates the task was accepted (NOT that the destination was reached!) |
| `task_id` | 服务确认的 task_id / task_id confirmed by the service |
| `info` | 失败时的错误描述字符串 / Error description string on failure |

### 5.3 Ctrl+C 优雅取消 (Graceful Cancellation with Ctrl+C)

```python
try:
    while True:
        state = get_nav_state(task_id)
        if state == "PncServiceState_SUCCESS":
            print("到达目标 / Destination reached")
            break
        time.sleep(1.0)

except KeyboardInterrupt:
    print("取消导航... / Cancelling navigation...")
    cancel_nav(task_id)
    print("已取消 / Cancelled")
```

这个模式很重要：用户按 Ctrl+C 时，不能让机器人继续走，必须明确发取消指令。

> *This pattern is critical: when the user presses Ctrl+C, the robot must not keep walking — you must explicitly send a cancel command.*

## 8. 常见问题 (Common Issues)

**Q: `导航下发失败: 机器人不在 RL_LOCOMOTION_DEFAULT 状态` / Navigation submission failed: robot not in RL_LOCOMOTION_DEFAULT**
A: 需要先切换动作模式。导航前确认 / Switch action mode first. Before navigating:
```python
if get_action() != "McAction_RL_LOCOMOTION_DEFAULT":
    set_action("McAction_RL_LOCOMOTION_DEFAULT")
    wait_for_action("McAction_RL_LOCOMOTION_DEFAULT")
```

**Q: 导航立刻变成 FAILED / Navigation immediately becomes FAILED**
A: 可能原因 / Possible causes:
- 目标点被障碍物遮挡 / Target blocked by an obstacle
- 机器人当前位置离地图边界太近 / Robot too close to the map boundary
- 重定位失败（机器人不知道自己在哪里）/ Relocalization failed (robot doesn't know its position)

**Q: 导航进入 PAUSED 一直不恢复 / Navigation stuck in PAUSED**
A: 机器人可能被障碍物卡住。人工清除障碍后任务会自动恢复，或手动取消重试。

> *The robot may be blocked by an obstacle. Clear the obstacle manually and the task will resume, or cancel and retry.*
